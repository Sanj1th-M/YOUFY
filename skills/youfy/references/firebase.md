## Initial Setup

### Step 1 — Create Firebase Project

1. Go to https://console.firebase.google.com
2. Click **Add project** → name it `youfy`
3. Disable Google Analytics (not needed)
4. Click **Create project**

### Step 2 — Enable Email/Password Auth

1. Console → **Authentication** → **Sign-in method**
2. Click **Email/Password** → Enable → Save

### Step 3 — Create Firestore Database

1. Console → **Firestore Database** → **Create database**
2. Choose **Start in production mode** (we set rules manually)
3. Select your region → Done

### Step 4 — Add Android App

1. Console → Project Settings → **Add app** → Android
2. Package name: `com.yourname.youfy`
3. Download `google-services.json` → place in `android/app/`

### Step 5 — Add iOS App

1. Console → Project Settings → **Add app** → iOS
2. Bundle ID: `com.yourname.youfy`
3. Download `GoogleService-Info.plist` → place in `ios/Runner/`

### Step 6 — Run FlutterFire CLI

```bash
# Install CLI
dart pub global activate flutterfire_cli

# Configure — auto-generates firebase_options.dart
flutterfire configure --project=your-firebase-project-id
```

This generates `lib/firebase_options.dart` — do NOT edit this file manually.

## Firebase Auth Flow (Full)

```
Flutter (login screen)
  ↓
FirebaseAuth.signInWithEmailAndPassword(email, password)
  ↓ success
FirebaseAuth returns User object + ID Token (valid 1 hour)
  ↓
Flutter stores nothing — token is managed by Firebase SDK automatically
  ↓
Every API call: user.getIdToken() → fresh token (auto-refreshed by SDK)
  ↓
Authorization: Bearer <token>  (in Dio interceptor)
  ↓
Node.js backend: admin.auth().verifyIdToken(token)
  ↓
Valid → req.user = { uid, email, ... } → proceed
Invalid → 401 Unauthorized
```

### Token Auto-Refresh (Flutter — already handled in api_service.dart)

```dart
// In Dio interceptor (lib/services/api_service.dart)
_dio.interceptors.add(InterceptorsWrapper(
  onRequest: (options, handler) async {
    final user = FirebaseAuth.instance.currentUser;
    if (user != null) {
      // getIdToken() returns cached token if valid, refreshes if expired
      final token = await user.getIdToken();
      options.headers['Authorization'] = 'Bearer $token';
    }
    handler.next(options);
  },
));
```

### Backend Token Verification (Node.js)

```javascript
// src/middleware/auth.js
const { auth } = require('../config/firebase');

async function verifyToken(req, res, next) {
  const header = req.headers.authorization;
  if (!header?.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'No token provided' });
  }
  try {
    const decoded = await auth.verifyIdToken(header.split('Bearer ')[1]);
    req.user = decoded; // { uid, email, name, ... }
    next();
  } catch {
    res.status(401).json({ error: 'Invalid or expired token' });
  }
}
```

### Firebase Admin SDK Setup (Node.js backend)

```javascript
// src/config/firebase.js
const admin = require('firebase-admin');

admin.initializeApp({
  credential: admin.credential.cert({
    projectId:   process.env.FIREBASE_PROJECT_ID,
    clientEmail: process.env.FIREBASE_CLIENT_EMAIL,
    // Replace escaped newlines — critical for private key
    privateKey:  process.env.FIREBASE_PRIVATE_KEY.replace(/\\n/g, '\n'),
  }),
});
```

### Getting Service Account Credentials

1. Console → Project Settings → **Service accounts**
2. Click **Generate new private key** → download JSON
3. Extract these 3 values into your `.env`:
   - `project_id` → `FIREBASE_PROJECT_ID`
   - `client_email` → `FIREBASE_CLIENT_EMAIL`
   - `private_key` → `FIREBASE_PRIVATE_KEY`

> ⚠️ NEVER commit the JSON file or the `.env` file to git.

## Firestore Data Structure

```
users/                                  (collection)
  {userId}/                             (document — userId = Firebase Auth UID)
    playlists/                          (sub-collection)
      {playlistId}/                     (document)
        name:       string
        createdAt:  timestamp
        songs:      array of SongRef

SongRef object shape:
{
  videoId:   string,   // YouTube video ID
  title:     string,
  artist:    string,
  thumbnail: string,   // URL to thumbnail image
  duration:  number    // seconds
}
```

> Songs are stored by reference (metadata only), NOT as full documents.
> Audio is always fetched live from the backend when played.

## Firestore Security Rules

Paste these in Console → Firestore → **Rules** tab:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users can only read/write their OWN playlists
    match /users/{userId}/playlists/{playlistId} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }

    // Deny everything else
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

**What these rules enforce:**
- A logged-in user can only access their own `users/{theirUID}/playlists/...`
- No user can read another user's playlists
- Unauthenticated requests are always rejected
- Any collection not explicitly listed is denied

> ❌ NEVER disable or weaken these rules, even for testing.
> Use the Firebase Emulator Suite for local development instead.

---

## Firestore Read/Write from Backend

The backend (Node.js) uses the Admin SDK which bypasses security rules.
Security rules only apply to client-side SDK calls (Flutter).
The backend enforces access control via the `verifyToken` middleware.

```javascript
// Only access the user's own playlists — enforced by uid from verified token
const ref = db.collection('users').doc(req.user.uid).collection('playlists');
```

## Common Firebase Auth Errors (Flutter)

```dart
try {
  await authService.signIn(email, password);
} on FirebaseAuthException catch (e) {
  switch (e.code) {
    case 'user-not-found':
      // No account with this email
    case 'wrong-password':
      // Incorrect password
    case 'invalid-email':
      // Bad email format
    case 'user-disabled':
      // Account disabled in Firebase console
    case 'too-many-requests':
      // Too many failed attempts — show "try again later"
    default:
      // Generic error
  }
}
```

## Local Development with Firebase Emulator

```bash
# Install Firebase CLI
npm install -g firebase-tools
firebase login

# Initialize emulators in project root
firebase init emulators
# Select: Authentication, Firestore

# Start emulators
firebase emulators:start
```

Connect Flutter to emulator:

```dart
// In main.dart, before Firebase.initializeApp()
if (kDebugMode) {
  FirebaseAuth.instance.useAuthEmulator('localhost', 9099);
  FirebaseFirestore.instance.useFirestoreEmulator('localhost', 8080);
}
```
